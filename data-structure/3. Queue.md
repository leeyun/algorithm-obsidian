## 1. 큐에 대한 설명

큐는 데이터가 삽입된 순서대로 먼저 삭제되는 선입선출 구조를 가진 선형 자료구조다. 데이터의 삽입은 한쪽 끝에서, 삭제는 반대쪽 끝에서 일어난다.

* 선입선출 구조: 먼저 들어온 데이터가 먼저 나가는 방식으로 대기열이나 터널과 같은 형태를 띤다.
* 주요 연산: 데이터를 큐의 맨 뒤에 넣는 연산을 push, 맨 앞의 데이터를 꺼내는 연산을 pop이라고 한다. 맨 앞의 데이터를 확인만 하는 연산은 peek이라고 부른다.
* 장점: 데이터가 입력된 순서대로 정확하게 처리되므로 작업의 순서를 보장해야 하는 시스템에서 매우 유용하다. 양 끝에서 삽입과 삭제가 독립적으로 이루어지므로 연산 속도가 빠르다.
* 단점: 중간에 있는 데이터에 직접 접근하거나 검색할 수 없다. 고정된 크기의 선형 배열로 구현할 경우, 데이터를 삭제하면 앞부분에 빈 공간이 생겨 메모리가 낭비되며 이를 꽉 찬 것으로 잘못 인식할 수 있다. 이러한 문제는 배열을 원형으로 연결한 원형 큐를 사용하여 해결해야 한다.

## 2. 의사코드

큐의 핵심 연산인 데이터 삽입, 삭제, 확인에 대한 기본 논리를 보여주는 의사코드다.

```text
// 1. 데이터 삽입
function push(queue, value, rear, max_size):
    if rear == max_size - 1:
        return "큐 공간 가득 참 오류"
    
    rear = rear + 1
    queue[rear] = value

// 2. 데이터 삭제 및 반환
function pop(queue, front, rear):
    if front > rear:
        return "큐 비어있음 오류"
    
    value = queue[front]
    queue[front] = null
    front = front + 1
    return value

// 3. 맨 앞 데이터 확인
function peek(queue, front, rear):
    if front > rear:
        return "큐 비어있음 오류"
    
    return queue[front]
```

## 3. 시간 복잡도

큐는 삽입과 삭제가 양 끝에서 각각 독립적으로 이루어지므로 두 연산에 대해 매우 효율적이다.

| 연산 | 시간 복잡도 | 설명 |
| :--- | :--- | :--- |
| 삽입 | O(1) | 큐의 맨 뒤에 데이터를 바로 추가하므로 다른 요소를 이동시킬 필요가 없다. |
| 삭제 | O(1) | 큐의 맨 앞 데이터를 바로 제거하므로 다른 요소를 이동시킬 필요가 없다. |
| 확인 | O(1) | 맨 앞 요소의 위치를 항상 알고 있으므로 즉시 접근하여 확인한다. |
| 검색 | O(n) | 특정 데이터를 찾기 위해 맨 앞부터 원하는 값이 나올 때까지 하나씩 꺼내며 탐색해야 한다. |

## 4. C 언어 및 파이썬 예시

### C 언어

C 언어에서는 배열과 앞뒤 위치를 가리키는 변수 두 개를 활용하여 고정된 크기의 큐를 구현한다.

```c
#include <stdio.h>
#define MAX_SIZE 5

int queue[MAX_SIZE];
int front = 0;
int rear = -1;

// 데이터 삽입
void push(int value) {
    if (rear == MAX_SIZE - 1) {
        printf("큐 공간 가득 참\n");
        return;
    }
    queue[++rear] = value;
}

// 데이터 삭제 및 반환
int pop() {
    if (front > rear) {
        printf("큐 비어있음\n");
        return -1;
    }
    return queue[front++];
}

int main() {
    push(10);
    push(20);
    push(30);

    printf("pop된 값: %d\n", pop());
    printf("pop된 값: %d\n", pop());

    return 0;
}
```

### 파이썬

파이썬에서는 리스트를 사용할 수도 있지만, 성능상 유리한 내장 모듈인 deque를 사용하여 큐를 구현하는 것이 일반적이다.

```python
from collections import deque

# 큐 초기화
queue = deque()

# 데이터 삽입
queue.append(10) # push 연산과 동일한 역할
queue.append(20)
queue.append(30)

# 맨 앞 데이터 확인
if queue:
    print(f"맨 앞의 값: {queue[0]}") # peek 연산과 동일한 역할

# 데이터 삭제 및 반환
print(f"pop된 값: {queue.popleft()}") # pop 연산과 동일한 역할
print(f"pop된 값: {queue.popleft()}")

# 현재 큐 상태 확인
print(f"남은 큐: {list(queue)}")
```
